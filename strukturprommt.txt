# RadarChartToolkit – Projektübersicht

## Projektziel

Entwicklung eines modularen Python-Toolkits zum Erstellen, Bearbeiten, Importieren, Exportieren und Visualisieren von Radar Charts.

Der Fokus liegt auf:
- sauberer Architektur
- klarer Trennung der Verantwortlichkeiten
- guter Erweiterbarkeit
- einfacher Integration in andere Projekte (z.B. Godot)

Das Projekt soll nicht nur ein einzelnes Skript sein, sondern eine wiederverwendbare Bibliothek.

# Datenmodell (models.py)

Es existiert ein vollständiges Datenmodell auf Basis von Dataclasses.

## RadarDataset

Speichert einen einzelnen Datensatz.

```python
RadarDataset
├── name
└── values
```

---

## RadarData

Speichert sämtliche Diagrammdaten.

```python
RadarData
├── title
├── labels
└── datasets
```

---

## RadarStyle

Speichert sämtliche Darstellungseinstellungen.

Momentan vorhanden:

- frame
- line_color
- fill_color
- fill
- alpha
- ring_count
- frame_width
- grid_width
- label_distance
- scale_label_offset
- grid_alpha

Alle Styleparameter sind zentral an einer Stelle definiert.

---

## RadarChart

Kombiniert Daten und Style.

```python
RadarChart
├── RadarData
└── RadarStyle
```

# Plot-System (radar_plot.py)

Das Plotting wurde komplett modular aufgebaut.

Es existieren folgende Funktionen:

```
plot_chart()

│
├── _get_max_value()
├── _calculate_scale()
├── _compute_geometry()
├── _setup_axes()
├── _draw_frame()
├── _draw_grid()
├── _draw_labels()
└── _draw_datasets()
```

# Excel Import

Der Excel Import ist nahezu vollständig.

Aktuelle Struktur:

```
load_chart()

│
├── _load_meta()
├── _load_data()
└── _load_style()
```

Importiert:

Meta
- Titel

Data
- Labels
- mehrere Datensätze

Style
- sämtliche Styleparameter

Außerdem:

- Typprüfung
- Fehlerbehandlung
- None-Prüfungen
- numerische Validierung

# Excel Export

Der Export ist ebenfalls modular.

Aktuelle Struktur:

```
save_chart()

│
├── _save_meta()
├── _save_data()
└── _save_style()
```

Exportiert:

Meta

```
Property | Value
title    | ...
```

Data

```
Label | Dataset1 | Dataset2 | ...
```

Style

```
Property | Value
```

Das Excelformat entspricht dem Importformat.

# Architektur

Es wurde bewusst auf Modularität geachtet.

Jede Datei besitzt eine klar definierte Aufgabe.

```
models
        │
        ▼
excel_handler
        │
        ▼
RadarChart Objekt
        │
        ▼
radar_plot
```

Keine Funktion übernimmt mehrere Verantwortlichkeiten.

---

# Entwicklungsprinzipien

Während der Entwicklung gelten folgende Regeln:

- kleine Funktionen
- Single Responsibility Principle
- möglichst wenige Magic Numbers
- Typannotationen
- Dataclasses
- Fehler früh erkennen
- keine riesigen Funktionen
- leicht testbar
- leicht erweiterbar


## 2. Excel Validator

Vor dem Laden soll geprüft werden:

- existieren alle Sheets?
- stimmen die Namen?
- fehlen Stylewerte?
- fehlen Labels?
- Datasetlängen korrekt?
- numerische Werte gültig?

Dadurch entstehen verständliche Fehlermeldungen statt Python Exceptions.
